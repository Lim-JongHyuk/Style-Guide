B. 타이포그래피 (Typography System)

서체의 균일한 공간 설정을 위한 행간 정의와 웹 표준 배수 계산법, 그리고 브라우저 렌더링에 따른 레이아웃 바인딩 규칙을 정의합니다.

2-5 폰트 행간 및 타이포 공간 배수 가이드라인

가독성이 높은 타이포그래피의 핵심은 정보의 밀도를 제어하는 '균일한 공간 설정'에 있습니다. 본 디자인 시스템은 글자가 가지는 고유의 가상 면적을 기준으로 이상적인 행간 공식과 글로벌 웹 접근성 표준(WCAG 2.1)을 모두 만족하는 개발 규격을 선언합니다.

글자의 몸통과 면의 개념 (Body & Face Area)

글자 디자인 시스템에는 '면(Face)'과 '몸통(Body)'이라는 구조적 개념이 존재합니다. 이상적인 타이포그래피는 글줄사이, 글자사이, 낱말사이가 모두 시각적 면간격으로 통일되는 구조입니다. 글자의 몸통을 기준으로 단순 정량 행간을 적용하면 폰트 고유의 여백 차이로 인해 시각적으로 글보다 행간이 과도하게 넓어보이는 착시를 일으킵니다. 반면 글자의 가상 면적을 기준으로 공간을 설정하면 글줄과 행간의 비중이 균등하게 배치됩니다.

면적 기준 1.75 배수 법칙 및 WCAG 웹 접근성 수치

이러한 면적 기준의 규칙을 수학적 수치로 환산하면 가장 가독성이 높은 스펙인 1.75배수가 산출됩니다. 글로벌 웹 접근성 가이드라인 WCAG 2.1 규격("문단 내의 띄어쓰기(Space) 및 행간은 적어도 글자 크기의 1.5배 이상이어야 한다")을 근거로 시스템의 최소값은 1.5, 표준 권장 기본값은 1.75로 고정바인딩합니다. 만약 폰트 크기 대비 배수 결과값에서 소수점이 발생할 경우, 레이아웃의 픽셀 그리드 깨짐을 방지하기 위해 반올림(Round)한 정수 수치를 최종 렌더링 값으로 적용합니다.

행간 설정별 가독성 및 가이드라인 준수 비교 (Line-height Comparison)
좁은 행간 (FAIL - line-height: 1.15)

글자의 몸통만을 기준으로 행간을 좁게 설정한 나쁜 예시입니다. 글줄 사이의 간격이 면적 기준을 만족하지 못하여 위아래 글자가 서로 겹쳐 보이며, 사용자가 문단을 읽을 때 다음 줄로 시선이 자연스럽게 이동하지 못하고 가독성이 극도로 저하되는 현상이 발생합니다.

글자 크기 비례 밀도 높음 WCAG 규격 미달
권장 표준 행간 (PASS - line-height: 1.75)

면적 기준 규칙과 디자인 시스템 권장 배수인 1.75를 올바르게 적용한 가독성 모범 사례입니다. 글줄과 행간의 균형이 완벽히 일치하여 시각적으로 안정감을 주며, 저시력자나 고령의 사용자도 긴 단락의 문장 콘텐츠를 왜곡 없이 명확하게 읽어 내려갈 수 있습니다.

CSS 브라우저 렌더링 메커니즘 (Half-Leading 여백 유의사항)

일반적인 2D 그래픽 툴(Figma, Photoshop 등)은 텍스트 베이스라인 아래를 기준으로 행간 여백을 쌓아가지만, 웹 브라우저(CSS) 엔진은 설정된 행간 값과 서체 고유 크기의 차이만큼 여백을 계산하여 위와 아래로 정확히 1/2씩 나누어 분할 적용합니다. 이를 Half-Leading이라고 합니다. 따라서 컴포넌트에 마진(Margin)이나 패딩(Padding) 값을 설정할 때, 서체 위아래에 숨겨진 공백 행간 값이 합산되어 레이아웃이 밀려나는 현상이 발생하므로 하이파이 코딩 시 이를 반드시 계산 공식에 고려해야 합니다.

브라우저 상·하단 1/2 행간 분할 박스 모델 (CSS Line Box Box-Model)
위쪽 여백: Half-Leading Space [ (line-height - font-size) / 2 ]
Actual Text Content Area (글자 실질 폰트 영역)
아래쪽 여백: Half-Leading Space [ (line-height - font-size) / 2 ]

프로덕션 시스템 행간 토큰 레지스트리

타이포그래피의 계층 구조별 용도에 맞춰 자동 연산 바인딩되는 디자인 시스템 표준 line-height 토큰 매트릭스입니다.

시스템 매칭 토큰 데이터 (Line-height Token Scale)
Token Name Scale Value 의도된 UI/UX 타이포그래피 용도 정수 연산 적용 예시 (반올림 규칙)
--lh-heading 1.3 대형 헤드라인, 24px 이상의 타이틀 (자체 크기가 커 행간 압축) 32px 폰트 × 1.3 = 41.6px → 42px
--lh-subheading 1.5 서브 타이틀, 중형 컴포넌트 제목 (WCAG 접근성 최소 기준 통과) 20px 폰트 × 1.5 = 30px → 30px
--lh-main-body 1.75 디자인 시스템 권장 기본값, 일반 본문 리딩 단락 전체 16px 폰트 × 1.75 = 28px → 28px
--lh-caption 1.6 데이터 표 내부 텍스트, 캡션, 메타 데이터 등 소형 서체 단락 13px 폰트 × 1.6 = 20.8px → 21px
디자이너 & 엔지니어 협업 코멘트 (Engineering Check)
CSS로 서체를 마크업할 때는 고정 픽셀(px) 단위의 line-height 선언 대신, 단위가 없는 상대 배수 수치(line-height: 1.75;)를 사용하는 것을 원칙으로 합니다. 단위를 생략해야만 자식 요소들이 부모의 고정 픽셀 값을 상속받지 않고, 자신의 바인딩된 고유 font-size에 비례하여 유연하고 안전하게 계산된 행간을 재렌더링할 수 있습니다.
