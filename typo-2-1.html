B. 타이포그래피 (Typography System)

다양한 디스플레이 출력 장치에서 서체의 가독성을 완벽히 제어하기 위한 CSS 크기 단위 체계를 확립합니다.

2-1 폰트 단위 개요 및 개념 정의

디지털 화면 인터페이스를 설계할 때 타이포그래피의 크기를 선언하는 단위는 결과물의 선명도와 반응형 유연성을 결정짓는 핵심 기반입니다. 실무 웹/앱 환경에서 사용하는 4대 핵심 단위의 물리적 개념 정의는 아래와 같습니다.

pixel

px

디스플레이 화면을 구성하는 가장 기본적인 최소 단위(1dot)입니다. 하나의 글꼴이 출력될 때 차지하는 물리적인 격자 점의 총개수를 명확하게 통제합니다.

point

pt

전통적인 타이포그래피 인쇄 영역에서 사용하는 단위로, 1포인트는 1/72인치에 해당합니다. 디스플레이 스크린이 발명되기 전 물리 종이 인쇄 시절부터 확립된 절대 크기 기준입니다.

em

em

상위 부모(Parent) 요소의 폰트 크기를 기준으로 비례하는 상대 단위입니다. 1em은 100%를 의미하며, 컴포넌트 내부에서 유기적으로 결합된 텍스트 크기를 변환할 때 유용합니다.

root em

rem

부모 요소가 아닌 최상위 루트(html) 태그의 기준 서체 크기(웹 표준 디폴트 16px)를 다이렉트로 추종하는 상대 배율 단위입니다.

웹 환경에서의 서체 단위 매커니즘: pt와 px의 차이

pt와 px의 가장 지대한 차이점은 최종 출력물이 표현되는 '장치(Media)'의 환경에서 극명하게 나타납니다.

최종 출력 대상이 물리적인 '종이'일 때는 화면의 밀도(ppi) 환경과 무관하게(72ppi, 96ppi, 150ppi) 24pt 서체를 사용하면 인쇄된 결과물 내에서 항상 자로 잰 듯 똑같은 물리 크기로 정확히 고정되어 인쇄됩니다.

반면, 출력 대상이 '디스플레이 장치'가 되면 기기 해상도 밀도(ppi)에 따라 화면상에 그려지는 24pt의 크기가 제각각 왜곡되어 다르게 보이는 기술적 하자가 발생합니다. 이때 화면 밀도의 간섭을 받는 pt 대신 물리 격자 단위인 px(Pixel)을 사용하여 서체를 24px로 명시해 주면, 모든 스마트폰과 모니터 브라우저는 기기 밀도와 상관없이 정확히 텍스트의 수직 높이를 물리 24픽셀 격자 크기로 깔끔하게 통일해 표시합니다. 따라서 어떤 브라우저에서도 일관된 UI 결과물을 도출해야 하는 웹 프로퍼티 개발 시에는 pt 단위 대신 px 단위를 사용할 것을 절대적으로 권장합니다.

동일한 24pt가 디스플레이 밀도(PPI)에 따라 왜곡 렌더링되는 실증 예시
72 ppi 환경과거 표준 모니터 밀도
1px * (72 / 72) = 100%
24pt 서체 크기 (표준 1:1)
96 ppi 환경일반 Windows 데스크톱 밀도
1px * (96 / 72) = 133%
24pt 서체 크기 (33% 확대 왜곡)
300 ppi 환경고밀도 플래그십 디바이스
1px * (300 / 72) = 416%
24pt 서체 크기 (416% 비대화 왜곡)
주니어 디자이너가 흔히 겪는 툴 주의 사항 안내
웹 브라우저 시스템이 연산하는 pt 단위와 포토샵(Photoshop) 같은 그래픽 디자인 툴의 구형 pt 단위 판정 기준은 다릅니다. 만약 포토샵/일러스트레이터 환경에서 텍스트 단위를 실수로 pt로 지정해 두고 수치를 기입하면, 해당 서체의 픽셀 변환 높이가 고스란히 픽셀(px) 크기로 매핑되어 출력되는 혼선이 일어납니다. UI 전용 설계 툴(Figma 등)에서는 항상 기본 단위가 픽셀 구조(1배수 dp)임을 명확히 인지하세요.

전문가를 위한 실무 폰트 선언 가이드 (px vs rem)
현재 모던 UI/UX 퍼블리싱 실무에서는 일반 본문 서체 설계 시 고정치수인 px 대신 최상위 루트 크기를 따르는 상대치수인 rem 단위를 결합해 사용하는 추세입니다. 고령자나 약시를 가진 사용자가 브라우저 설정에서 '글자 크기 크게 보기' 기능을 활성화했을 때, px로 고정된 웹사이트는 본문이 커지지 않아 접근성 인증에 실패하게 됩니다. 반면 rem 단위로 유연하게 코딩된 타이포 시스템은 사용자의 기기 설정에 맞춰 본문이 부드럽게 스케일업되므로 스마트한 반응형 인터페이스를 완성해 줍니다.
