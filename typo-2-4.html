B. 타이포그래피 (Typography System)

디자인 시스템 전반에 적용되는 서체의 컬러 운용 기준과 웹 접근성(Web Accessibility) 명도 대비 규격을 정의합니다.

2-4 폰트 컬러 및 웹 접근성 명도 대비 규칙

웹 사이트에서 제공하는 모든 텍스트 콘텐츠는 사용자의 장애 유무, 연령, 디바이스 환경에 관계없이 명확하게 인식될 수 있어야 합니다. 본 가이드는 글로벌 웹 표준(WCAG 2.1) 및 한국형 웹 콘텐츠 접근성 지침(KWCAG 2.1)을 기반으로 한 폰트 컬러 최적화 가이드라인입니다.

웹 접근성(Web Accessibility)의 정의

웹 접근성이란 신체적 장애나 인지적 제약이 있는 사용자를 포함하여, 모든 사용자가 웹 사이트에서 제공하는 정보와 기능을 동등하게 인지하고 이용할 수 있도록 보장하는 것을 뜻합니다. 접근성 가이드라인에서 '적합성 수준'이란 세부 지침들이 갖는 중요도와 우선순위에 따라 단계를 분류하여 타겟 기준을 충족하도록 관리하는 스펙 지표를 의미합니다.

명도 대비 및 등급별 스펙 축 (Contrast Ratio Scale)
1:1 측정 불가 (최저)
3:1 큰 텍스트 허용
4.5:1 일반 텍스트 최소 기준 (AA)
21:1 최고 대비 (Pure Black/White)

Level 1. 최소 완화 대비 (3:1)

큰 텍스트 단락(24px 이상 또는 18.5px 이상의 Bold 서체) 및 확대 가능한 인터페이스 컴포넌트에 한해 제한적으로 허용되는 최저 명도 수치입니다.

Level 2. 일반 표준 대비 (4.5:1)

기본 본문 및 일반 크기 텍스트의 필수 표준 수치입니다. 저시력자, 노인, 색각 이상자가 콘텐츠를 왜곡 없이 인지하는 최소 안전선입니다.

Level 3. 고대비 강화 스펙 (7:1 이상)

가장 이상적이고 높은 가독성을 제공하는 고대비 영역입니다. 주요 핵심 데이터, 가독성 강화가 필수적인 정보 텍스트 컴포넌트에 적극 권장됩니다.

텍스트 콘텐츠의 세부 명도 대비 지침

텍스트 디자인 시 배경 컬러 크래시를 방지하기 위해 다음 3가지 핵심 데이터 룰바인딩을 엄격히 적용합니다. (※ 계산식 기준: 1pt = 1.333px)

폰트 크기 및 두께별 명도대비 바인딩 맵 (Contrast Specification Matrix)
적용 대상 유형 서체 스펙 상세 조건 (크기/굵기) 필수 최소 명도 대비 가이드 라인 검증 결과 지표
일반 텍스트 콘텐츠 약 24px (18pt) 미만 또는 굵은 폰트 18.5px (14pt) 미만인 서체 전체 4.5 : 1 이상 AA 등급 필수 통과
대형 타이틀 및 Bold 서체 약 24px (18pt) 이상의 일반 서체 또는 약 18.5px (14pt) 이상의 굵은(Bold) 서체 3 : 1 까지 완화 가능 완화 기준 적용 영역
화면 확대 가능 UI 콘텐츠 사용자가 임의로 화면을 확대 가동할 수 있도록 접근성 기능을 구현한 텍스트 3 : 1 까지 완화 가능 반응형 확대 장치 링크 필요

합격/불합격 시각 인터페이스 비교 목업 (Pass / Fail UI Mockups)

배경색과 폰트 컬러의 매칭 오류로 인해 웹접근성이 저해되는 실제 예시와 표준 디자인 시스템 토큰을 적용하여 합격한 레이아웃 스펙을 실무 시뮬레이션 화면으로 비교합니다.

명도 대비 실무 매칭 검증 모듈 (Contrast Ratio Validator)
접근성 규격 미달 (FAIL)

공지사항 안내 데이터

본문 텍스트 명도 대비가 너무 낮아 저시력자나 야간 모드 사용자가 글자를 올바르게 인지하기 어렵습니다.

배경 #f8fafc 대비 폰트색 #cbd5e1 명도비 1.48 : 1
웹접근성 통과 표준 (PASS)

공지사항 안내 데이터

디자인 시스템 전사 가이드를 준수하여 4.5:1 이상의 명백한 명도차를 확보함으로써 최상의 가독성을 제공합니다.

배경 #ffffff 대비 폰트색 #475569 명도비 5.17 : 1

엔지니어링 코멘트 (Accessibility Checking)
프로덕션 배포 전 폰트 컬러를 설정할 때는 반드시 배경색상의 Hex 값과 대조하여 자동화 린터 혹은 크롬 개발자 도구(Lighthouse Audit)의 Contrast 검증기로 최소 4.5:1 이상(대형 서체는 3:1 이상)이 구현되었는지 사전 스크리닝을 진행하는 것을 원칙으로 합니다.
